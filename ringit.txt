## IMPORTANT DECISION: Vapi vs Retell AI

Before you build, understand the difference:

| Feature | Vapi | Retell AI |
|---------|------|-----------|
| **Setup complexity** | Easier, more polished UI | Slightly more technical |
| **Phone numbers** | Built-in (Twilio integration) | You manage Twilio separately |
| **Prompt generation** | You write or use Claude | You write prompts |
| **Voice quality** | Good, multiple voices | Excellent, very natural |
| **Customization** | Limited but intuitive | More flexible |
| **Webhooks** | ✅ Yes | ✅ Yes |
| **Cost** | $0.25-0.75/min approx | $0.10-0.30/min approx |
| **Best for** | Non-technical founders | Developers who want control |
| **API Maturity** | Well-documented | Well-documented |

**Recommendation for Ringit MVP:** **Vapi** — easier setup, faster to launch. Retell if you want more customization or cost optimization.

This document covers **Vapi** as primary. Retell can be swapped in (nearly identical architecture).

## PART 0 — Understanding Vapi

### What is Vapi?

Vapi is a platform that handles voice AI conversations. Think of it like a "hosted voice receptionist service."

You give Vapi:
1. A phone number (via Twilio)
2. A prompt (the agent's personality)
3. A webhook URL (to get call events)

Vapi handles:
- Speech-to-text (understands caller)
- LLM inference (decides what to say)
- Text-to-speech (speaks the response)
- Call routing (when someone calls the number)
- Call logs and transcripts
- Webhooks back to your server

### Key Difference from Atoms

- **Atoms:** Smallest AI owns everything, you manage multi-tenant complexity
- **Vapi:** You manage the prompt, Vapi just runs conversations. Simpler but less "magic."

## PART 3 — The User Experience (Complete Flow)

Same as the Atoms version, but with Vapi powering the voice conversations.

### Scene 1: Priya Lands on Ringit

Landing page. Clicks "Get Started." Creates account via Supabase Auth.

### Scene 2: The 4-Question Wizard

**Step 1:** Industry (Dental)
**Step 2:** Business details (name, services)
**Step 3:** Tone (warm & friendly)
**Step 4:** Lead email

Clicks **"Build My Agent"**

### Scene 3: The Build Screen (What Happens Behind)

Your `/api/provision` endpoint:

```typescript
// Step A: Write the agent prompt (manually or via Claude)
// Step B: Create assistant on Vapi
// Step C: Buy phone number from Twilio
// Step D: Assign number to Vapi assistant
// Step E: Register webhook with Vapi
// Step F: Save to Supabase
```

Total time: ~20 seconds.

### Scene 4: Success + Optional Voice Clone

Shows phone number. Optional: "Record your voice" (voice clone feature - post-MVP).

### Scene 5: Agent Chat Editor

URL: `ringit.ai/agent/[id]`

User types: *"Make it warmer and more conversational"*

Your `/api/edit-assistant` endpoint:
- Gets current prompt
- Calls Claude to rewrite it with the user's request
- Updates Vapi assistant with new prompt
- Live in ~5 seconds

### Scene 6: Real Call

Patient calls the number → Vapi assistant answers → calls captured as leads → sent to Priya's email.

### Scene 7: Dashboard

Shows calls, leads, transcripts. All data cached from Vapi webhooks in Supabase.

```typescript
      voiceMethod: 'POST',
    });
    console.log('✅ Twilio number configured for SIP routing to Vapi');

    // ─── STEP E: Register webhook with Vapi ───────────────────────

    await vapiClient.post('/webhook', {
      url: `${process.env.NEXT_PUBLIC_BASE_URL}/api/webhooks/vapi`,
      events: ['call_started', 'call_ended', 'transcript_ready'],
    });
    console.log('✅ Webhook registered with Vapi');

    const { data: savedAgent } = await supabase.from('agents').insert({
      user_id: user.id,
      business_name: businessName,
```

## PART 5 — Twilio → Vapi SIP Routing (The Tricky Part)

When someone calls your Twilio number, Twilio needs to forward it to Vapi. There are two approaches:

### Approach A: Vapi's Twilio Integration (Easiest)

Vapi has a native Twilio integration. In this flow:

1. You buy the number on Twilio
2. You connect your Twilio account to Vapi (OAuth or API key)
3. Vapi can assign phone numbers directly

```typescript
// Contact Vapi support to link your Twilio account
// Once linked, you can do this:

await vapiClient.post('/assistant', {
  name: `${businessName} Receptionist`,
  voiceId: 'en_US_female_1',
  model: { /* ... */ },
  // VAPI ASSIGNS THE PHONE NUMBER DIRECTLY
  phoneNumbers: [phoneNumber] // Pass in the Twilio number
});
```

**Pros:** Simplest setup
**Cons:** Requires OAuth setup with Vapi

### Approach B: SIP Trunking (More Control)

Twilio forwards calls to Vapi via SIP protocol.

```typescript
// 1. Create a SIP trunk in Twilio
const trunk = await twilioClient.api.v1.sip.trunks.create({
  friendlyName: 'Vapi SIP Trunk',
  domainName: `ringit-${process.env.TWILIO_ACCOUNT_SID}.sip.twilio.com`
});

// 2. Add a termination URI (where calls go)
await twilioClient.api.v1.sip.trunks(trunk.sid).terminationSettings.update({
  uri: `sip:${process.env.VAPI_SIP_ENDPOINT}`,
});

// 3. Update the Twilio phone number to use this trunk
await twilioClient.incomingPhoneNumbers(twilioPhoneSid).update({
  trunkSid: trunk.sid
});
```

**Pros:** More control, explicit routing
**Cons:** Requires SIP configuration

## PART 6 — Edit Agent Endpoint (Chat Editor)

When user types "make it warmer," this endpoint fires:

```typescript
// /app/api/edit-assistant/route.ts

export async function POST(req: NextRequest) {
  const { agentId, userMessage } = await req.json();

  // 1. Get current agent from Supabase (for context)
  const { data: agent } = await supabase
    .from('agents')
    .select('vapi_assistant_id, current_prompt, industry')
    .eq('id', agentId)
    .single();

  // 2. Use Claude to rewrite the prompt based on user feedback
  const claude = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });
  
  const editResponse = await claude.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 2000,
    messages: [{
      role: 'user',
      content: `Here is a ${agent.industry} receptionist prompt:

${agent.current_prompt}

The user wants to make this change: "${userMessage}"

Return the COMPLETE updated prompt. Keep the structure the same, just apply the requested change.
Return ONLY the prompt text, no explanation.`
    }]
  });

  const newPrompt = (editResponse.content[0] as any).text;

  // 3. Update the Vapi assistant with the new prompt
  await vapiClient.patch(`/assistant/${agent.vapi_assistant_id}`, {
    model: {
      messages: [
        {
          role: 'system',
          content: newPrompt
        }
      ]
    }
  });

  // 4. Save the updated prompt to Supabase
  await supabase
    .from('agents')
    .update({ current_prompt: newPrompt })
    .eq('id', agentId);

  return NextResponse.json({
    message: `Done! Your agent is now: ${userMessage.toLowerCase()}. Changes are live.`,
    success: true
  });
}
```

## PART 7 — Webhook Handler (How Calls Come Back)

Vapi fires webhooks when calls end. Your server receives them:

```typescript
// /app/api/webhooks/vapi/route.ts

export async function POST(req: NextRequest) {
  const body = await req.json();
  const { event, data } = body;

  if (event === 'call_ended') {
    // Find which agent this call belongs to
    const { data: agent } = await supabase
      .from('agents')
      .select('id, user_id, lead_email, business_name')
      .eq('vapi_assistant_id', data.assistantId)
      .single();

    if (!agent) return NextResponse.json({ ok: true });

    // 2. Get the full call transcript from Vapi
    const callDetails = await vapiClient.get(`/call/${data.callId}`);
    const { transcript, duration, summary } = callDetails.data;

    // 3. Extract lead info from transcript
    // (AI-based: use Claude to parse "What's the caller's name, phone, and what did they want?")
    const leadInfo = await extractLeadInfo(transcript);

    // 4. Save to Supabase
    await supabase.from('calls').insert({
      agent_id: agent.id,
      user_id: agent.user_id,
      vapi_call_id: data.callId,
      duration_seconds: duration,
      transcript: transcript,
      summary: summary,
      lead_name: leadInfo.name,
      lead_phone: leadInfo.phone,
      lead_intent: leadInfo.intent,
      created_at: new Date().toISOString()
    });

    // 5. Send lead email to the business owner
    await sendLeadEmail({
      to: agent.lead_email,
      businessName: agent.business_name,
      callDuration: duration,
      leadName: leadInfo.name,
      leadPhone: leadInfo.phone,
      leadIntent: leadInfo.intent,
      transcript: transcript,
    });

    // 6. POST to ServiceTitan (if customer has it set up) - POST-MVP
    if (agent.servicetitan_webhook_url) {
      await fetch(agent.servicetitan_webhook_url, {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          name: leadInfo.name,
          phone: leadInfo.phone,
          serviceType: leadInfo.intent,
          notes: transcript.substring(0, 500) // first 500 chars
        })
      }).catch(err => console.error('ServiceTitan sync failed:', err));
    }
  }

  return NextResponse.json({ ok: true });
}

// Helper: Extract lead info from transcript using Claude
async function extractLeadInfo(transcript: string) {
  const claude = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY });
  
  const response = await claude.messages.create({
    model: 'claude-sonnet-4-20250514',
    max_tokens: 200,
    messages: [{
      role: 'user',
      content: `Extract from this call transcript:
1. Caller's name (or "Unknown")
2. Caller's phone (or "Not provided")
3. What they wanted (brief, one line)

Transcript:
${transcript}

Return JSON only: { "name": "...", "phone": "...", "intent": "..." }`
    }]
  });

  const text = (response.content[0] as any).text;
  return JSON.parse(text);
}
```

```typescript
// When lead is captured, POST to ServiceTitan API

const sendToServiceTitan = async (leadInfo: {
  name: string;
  phone: string;
  intent: string;
  transcript: string;
}, vapiCallId: string) => {
  
  // ServiceTitan expects a specific format
  const servicetitanPayload = {
    firstName: leadInfo.name.split(' ')[0],
    lastName: leadInfo.name.split(' ')[1] || '',
    phone: leadInfo.phone,
    email: '', // not always available
    serviceType: leadInfo.intent,
    source: 'ringit_ai_receptionist',
    notes: leadInfo.transcript.substring(0, 500),
    tags: ['ringit', vapiCallId]
  };

  // Call their Create Lead API
  const response = await fetch('https://api.servicetitan.com/v2/companies/{companyId}/leads', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${customerServiceTitanToken}`, // from customer's account
      'Content-Type': 'application/json'
    },
    body: JSON.stringify(servicetitanPayload)
  });

  return response.json();
};
```

## PART 15 — Critical Things to Know

1. **Vapi's Twilio integration is easier.** Let Vapi manage phone numbers directly instead of SIP trunking.

2. **Use Retell instead of Vapi if cost matters.** Same features, 1/3 the price.

3. **Prompt engineering is NOW your job.** Unlike Atoms, Vapi doesn't auto-generate. You write the prompts.

4. **Claude is required for chat editor.** You MUST have Anthropic API key for the "edit agent" feature.

5. **Lead extraction from transcript is manual or AI-based.** Vapi gives you the transcript, you parse it.

6. **SIP configuration is optional.** Use Vapi's built-in Twilio integration instead.

7. **Retell is equally good but cheaper.** Swap it in seamlessly — same architecture.

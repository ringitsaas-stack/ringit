# RINGIT - FINAL TECHNICAL ARCHITECTURE (V1)

## Project Goal

Ringit is a multi-tenant AI Receptionist SaaS platform.

Businesses can:

* Sign up
* Create AI receptionists
* Get phone numbers
* Receive calls
* Capture leads
* View analytics
* Manage subscriptions

The platform must support multiple AI providers.

Initial provider:

* Retell AI (Preferred)

# Architecture Rules

Follow:

* SOLID Principles
* DRY
* KISS
* Feature Based Architecture

Never place business logic inside React components.

Never place database queries inside UI.

Never call Retell APIs directly from React components.

# Database Schema

## profiles

id UUID PK

email TEXT

full_name TEXT

created_at TIMESTAMP

updated_at TIMESTAMP

---

## agents

id UUID PK

user_id UUID FK

provider TEXT

provider_agent_id TEXT

business_name TEXT

industry TEXT

services TEXT

tone TEXT

lead_email TEXT

phone_number TEXT

phone_sid TEXT

current_prompt TEXT

status TEXT

created_at TIMESTAMP

updated_at TIMESTAMP

---

## calls

id UUID PK

agent_id UUID FK

provider_call_id TEXT

caller_phone TEXT

duration_seconds INTEGER

summary TEXT

transcript TEXT

created_at TIMESTAMP

---

## leads

id UUID PK

agent_id UUID FK

name TEXT

phone TEXT

intent TEXT

summary TEXT

status TEXT

created_at TIMESTAMP

---

## subscriptions

id UUID PK

user_id UUID FK

stripe_customer_id TEXT

stripe_subscription_id TEXT

plan TEXT

status TEXT

created_at TIMESTAMP

---

## webhook_events

id UUID PK

event_type TEXT

created_at TIMESTAMP

# Row Level Security

Users can only access:

their agents

their calls

their leads

their subscription

Example:

auth.uid() = user_id

Enable RLS on all tables.
